💳 Overview — What is Stripe?

Stripe is the world's most popular payment platform, used by companies like Amazon, Shopify, and Airbnb. It allows you to accept credit cards, debit cards, and Apple Pay / Google Pay securely on your website — without ever touching sensitive card data.

Stripe handles all the complex security (PCI compliance, encryption, fraud detection) for you. Your website simply sends a payment token to Stripe's servers — no card numbers are ever stored on your site.

🔒
Bank-Level Security
PCI DSS Level 1 certified. Stripe holds card data — your site never does.

Pricing
2.9% + 30¢ per successful card charge. No monthly fees. No setup costs.

Instant Payouts
Money lands in your bank account in 2 business days (can be same-day with Instant Payouts).

1 Create Your Stripe Account
  • Go to stripe.com and sign up
    Visit https://stripe.com and click "Start now". Fill in your email address, full name, and a secure password. Stripe is free to create an account — you only pay the 2.9% + 30¢ fee when you successfully charge a customer.
  • Verify your email address
    Stripe will send you a confirmation email. Click the link inside to verify your account.
  • Complete your business profile
    Stripe will ask for:
    • Business type — Select "Individual / sole trader" if you're running the studio as yourself, or "Limited company" if you've registered a company.
    • Business name — "Beyond the Canvas Art Studio" or "Farnaz Amin"
    • Industry — Arts & Crafts / Education
    • Your bank account details — for receiving payouts
  • Set up Two-Factor Authentication (strongly recommended)
    Go to Settings → Profile → Two-step authentication. Enable it with your phone. This protects your account and your money.
2 Get Your API Keys

API keys are how your website proves to Stripe that it's authorised to process payments on your account. You need two types:

Key Name Prefix Where It's Used Can It Be Public?
Publishable Key pk_test_ or pk_live_ Your HTML/JavaScript file (frontend) ✅ Yes
Secret Key sk_test_ or sk_live_ Your server/backend only — NEVER in HTML 🚫 NEVER
🚨
CRITICAL SECURITY WARNING
NEVER put your Secret Key (sk_live_...) in your HTML file, JavaScript, or anywhere a website visitor could see it. This would allow anyone to steal money from your account. The Secret Key must ONLY live on your server/backend, set as an environment variable.
  • Go to your Stripe Dashboard
    Log in at dashboard.stripe.com
  • Navigate to Developers → API Keys
    You'll see two keys: a pk_test_... (Publishable) and a sk_test_... (Secret). While testing, use the test keys. They look identical but start with test instead of live.
  • Copy and save your Publishable Key
    Copy the pk_test_... key — you'll need to paste it into your HTML. Keep the Secret Key safe for your server setup.
3 Add Stripe.js to Your Booking Page

Open your instructors.html file in a text editor (Notepad, VS Code, etc.). Find the </body> tag near the very bottom of the file. Add the Stripe script just before that closing tag:

HTML — Add to instructors.html before </body>

Stripe.js is a JavaScript library that creates a secure card input field on your form. It loads from Stripe's servers (not yours), which is part of how it stays secure and PCI compliant.

4 Update the Frontend JavaScript (Your HTML File)

In instructors.html, find the existing <script> block near the bottom. You need to replace the existing checkout function with the real Stripe version below.

🔍
Search Tip
In your text editor, press Ctrl+F (or Cmd+F on Mac) and search for handleCheckout — that's the function you need to replace.
JavaScript — Replace your handleCheckout section in instructors.html 
// ── STRIPE SETUP ─────────────────────────────────────────────
// Replace YOUR_PUBLISHABLE_KEY with your actual pk_test_... or pk_live_... key

const stripe = Stripe('YOUR_PUBLISHABLE_KEY'); // e.g. 'pk_test_51ABC...'
const elements = stripe.elements();

// Mount Stripe's secure card element into the #card-element div
const cardElement = elements.create('card', {
  style: {
    base: {
      fontFamily: 'Inter, sans-serif',
      fontSize: '15px',
      color: '#1A1A1A',
      '::placeholder': { color: '#888' }
    },
    invalid: { color: '#e53e3e' }
  }
});

cardElement.mount('#card-element'); // mounts the real card form

cardElement.on('change', ({ error }) => {
  const errorEl = document.getElementById('card-errors');
  errorEl.textContent = error ? error.message : '';
});

// ── CHECKOUT HANDLER ──────────────────────────────────────────
async function handleCheckout(e) {
  e.preventDefault();

  const fname = document.getElementById('firstName').value.trim();
  const lname = document.getElementById('lastName').value.trim();
  const email = document.getElementById('email').value.trim();
  const date  = document.getElementById('sessionDate').value;
  const time  = document.getElementById('sessionTime').value;

  if (!fname || !lname || !email || !date || !time) {
    alert('Please fill in all required fields.');
    return;
  }

  const btn = document.getElementById('submitBtn');
  btn.classList.add('loading');

  try {
    // 1. Ask your server to create a PaymentIntent
    const qty = parseInt(document.getElementById('qtyInput').value) || 1;
    const amountCents = currentPrice * qty * 100; // Stripe uses cents

    const response = await fetch('/create-payment-intent', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        amount: amountCents,
        currency: 'usd', // change to 'gbp', 'aud', 'cad' etc as needed
        metadata: {
          class: currentName,
          date: date,
          time: time,
          customer: `${fname} ${lname}`,
          email: email
        }
      })
    });

    const { clientSecret } = await response.json();

    // 2. Confirm the payment with Stripe using the card element
    const { error, paymentIntent } = await stripe.confirmCardPayment(clientSecret, {
      payment_method: {
        card: cardElement,
        billing_details: {
          name: `${fname} ${lname}`,
          email: email
        }
      }
    });

    if (error) {
      document.getElementById('card-errors').textContent = error.message;
      btn.classList.remove('loading');
      return;
    }

    // 3. Payment succeeded!
    document.getElementById('successRef').textContent =
      `Booking confirmed for ${fname}! Reference: ${paymentIntent.id}`;
    document.getElementById('successOverlay').classList.add('show');

  } catch (err) {
    alert('Payment failed. Please try again or contact us directly.');
    console.error(err);
    btn.classList.remove('loading');
  }
}

5 Set Up Your Backend (Server Side)

Stripe requires a server to securely create a "PaymentIntent" — a Stripe object that represents your payment request. Here are three options depending on how you host your website:

🟢 Option A — Netlify or Vercel Serverless Function (Easiest, Free)

If you host (or move) your site to Netlify or Vercel, you can add a free serverless function. No ongoing server costs. Great for small studios.

Netlify setup:

  • Create a netlify/functions/create-payment-intent.js file
    In your website folder, create this folder path and add the file below.
    JavaScript — netlify/functions/create-payment-intent.js
    // Install first: npm install stripe
const stripe = require('stripe')(process.env.STRIPE_SECRET_KEY);

exports.handler = async (event) => {
  try {
    const { amount, currency, metadata } = JSON.parse(event.body);

    const paymentIntent = await stripe.paymentIntents.create({
      amount,
      currency: currency || 'usd',
      metadata,
      automatic_payment_methods: { enabled: true }
    });

    return {
      statusCode: 200,
      headers: { 'Access-Control-Allow-Origin': '*' },
      body: JSON.stringify({ clientSecret: paymentIntent.client_secret })
    };
  } catch (err) {
    return {
      statusCode: 500,
      body: JSON.stringify({ error: err.message })
    };
  }
};
  • Add your Secret Key as an environment variable in Netlify
    In your Netlify dashboard, go to: Site settings → Environment variables → Add variable
    Key: STRIPE_SECRET_KEY
    Value: sk_test_... (paste your secret key — the one that starts with sk_)
  • Update the fetch URL in your HTML
    Change '/create-payment-intent' to '/.netlify/functions/create-payment-intent' in the JavaScript you added in Step 4.
🟡 Option B — Simple PHP Script (If Your Host Supports PHP)

Many web hosting providers (GoDaddy, Bluehost, Hostinger, cPanel hosts) support PHP by default. Create a PHP file on your server:

PHP — create-payment-intent.php (upload to your web server)
<?php
// Install Stripe PHP library via Composer: composer require stripe/stripe-php
// OR download from: https://github.com/stripe/stripe-php/releases

require_once 'vendor/autoload.php'; // path to Stripe library

header('Content-Type: application/json');
header('Access-Control-Allow-Origin: *');

\Stripe\Stripe::setApiKey('sk_test_YOUR_SECRET_KEY'); // ← PUT YOUR SECRET KEY HERE

$data = json_decode(file_get_contents('php://input'), true);

$paymentIntent = \Stripe\PaymentIntent::create([
  'amount'   => $data['amount'],
  'currency' => $data['currency'] ?? 'usd',
  'metadata' => $data['metadata'] ?? [],
  'automatic_payment_methods' => ['enabled' => true]
]);

echo json_encode(['clientSecret' => $paymentIntent->client_secret]);
?>

Then update your fetch URL to '/create-payment-intent.php' in the HTML JavaScript.

🔴 Option C — No-Code Alternative (Stripe Payment Links)

If you don't want to deal with code at all, you can use Stripe Payment Links. You create a payment link in your Stripe dashboard, and you simply change the "Book Now" buttons on your website to point to that link. Customers pay through Stripe's hosted page — no code needed on your site.

How to create a Payment Link: Stripe Dashboard → Payment Links → Create. Set the price, description, and Stripe handles everything.

6 Test Your Payments

Stripe provides test card numbers that simulate real payments without charging real money. Use these to make sure everything works before going live.

Card Number Expiry CVV What It Simulates

4000 0025 0000 3155 Any future date Any 3 digits 3D Secure authentication
4000 0000 0000 9995 Any future date Any 3 digits ❌ Declined — insufficient funds
4000 0000 0000 0069 Any future date Any 3 digits ❌ Declined — expired card
  • Go to your booking page and make a test booking
    Fill in all the form fields with test data. Use the card number 4242 4242 4242 4242, any future expiry date (e.g. 12/29), any 3-digit CVV, and any postcode.
  • Check your Stripe Dashboard for the payment
    Log in to dashboard.stripe.com → Payments. You should see the test payment appear within seconds. It will show a Test badge.
  • Test a declined card
    Use 4000 0000 0000 9995 to confirm your error handling works — customers should see a clear error message.
7 Go Live — Switch to Real Payments
⚠️
Only do this when you're ready to accept real money
Make sure you've tested thoroughly with the test keys first. Going live means real credit cards will be charged.
  • Complete Stripe account verification
    Stripe may ask you to verify your identity and bank account details before activating live payments. This usually takes 1–2 business days.
  • Get your Live API Keys
    In the Stripe Dashboard → Developers → API Keys, toggle from "Test" to "Live" and copy your live keys (pk_live_... and sk_live_...).
  • Update your Publishable Key in the HTML
    In instructors.html, replace pk_test_... with pk_live_... in the JavaScript.
  • Update your Secret Key environment variable
    In Netlify/your server, update the STRIPE_SECRET_KEY environment variable from sk_test_... to sk_live_....
  • Make a real £1/$1 test payment with your own card
    Book one slot with your real card and make sure the money appears in your Stripe balance. Then refund yourself from the Stripe Dashboard.
✅ Launch Checklist

Use this checklist to make sure you haven't missed anything before accepting real bookings:

  • Stripe account created and email verified
  • Bank account connected in Stripe for payouts
  • Identity verification completed in Stripe
  • Stripe.js script added to instructors.html
  • Publishable key added to JavaScript (never the Secret Key!)
  • Backend function / PHP script deployed and working
  • Secret key stored as environment variable on server (NOT in HTML)
  • Test payment processed successfully with card 4242 4242 4242 4242
  • Declined card error message displays correctly
  • Success overlay appears after successful payment
  • Switched to Live API keys for production
  • Confirmation email set up (optional but recommended — use Stripe's webhook to trigger this)
  • Refund policy added to booking page
